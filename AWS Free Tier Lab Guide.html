Passo 1: Configurazione Credenziali

A differenza di LocalStack, qui servono credenziali vere. Crea una Access Key dalla console IAM di AWS.

Nel terminale esegui:

aws configure

Inserisci:

  • AWS Access Key ID: (La tua chiave)
  • AWS Secret Access Key: (La tua chiave segreta)
  • Default region name: us-east-1
  • Default output format: json

Passo 3: Infrastructure as Code (Terraform)

Crea il file main.tf. Nota l'uso di random_id per garantire che il nome del bucket sia unico a livello globale.

terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
    random = {
      source = "hashicorp/random"
      version = "~> 3.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
  # Usa automaticamente le credenziali di 'aws configure'
}

# Generatore stringa casuale per nome bucket unico
resource "random_id" "suffix" {
  byte_length = 4
}

# 1. Bucket S3
resource "aws_s3_bucket" "my_bucket" {
  bucket        = "lab-freetier-${random_id.suffix.hex}"
  force_destroy = true 
}

# 2. Zip Codice
data "archive_file" "lambda_zip" {
  type        = "zip"
  source_file = "hello.py"
  output_path = "lambda_function.zip"
}

# 3. Ruolo IAM
resource "aws_iam_role" "iam_for_lambda" {
  name = "iam_for_lambda_freetier"

  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Action = "sts:AssumeRole"
        Effect = "Allow"
        Principal = {
          Service = "lambda.amazonaws.com"
        }
      },
    ]
  })
}

# Policy per i Log (CloudWatch)
resource "aws_iam_policy_attachment" "lambda_logs" {
  name       = "lambda_logs"
  roles      = [aws_iam_role.iam_for_lambda.name]
  policy_arn = "arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole"
}

# 4. Funzione Lambda
resource "aws_lambda_function" "test_lambda" {
  filename      = "lambda_function.zip"
  function_name = "hello_aws_real"
  role          = aws_iam_role.iam_for_lambda.arn
  handler       = "hello.lambda_handler"
  runtime       = "python3.9"
  source_code_hash = data.archive_file.lambda_zip.output_base64sha256
}

output "bucket_name" {
  value = aws_s3_bucket.my_bucket.id
}

output "lambda_name" {
  value = aws_lambda_function.test_lambda.function_name
}

Passo 5: Deploy e Test

1. Deploy

terraform init
terraform apply --auto-approve

Attendi circa 30-60 secondi per la creazione delle risorse su AWS.

2. Test (AWS CLI)

Nota: Non usiamo pi√π --endpoint-url.

# Leggi la risposta
cat response.json

Passo 7: PULIZIA (Fondamentale)

Per evitare costi futuri, distruggi l'infrastruttura immediatamente dopo aver finito il test.

terraform destroy --auto-approve

Vai nella console web di AWS (S3 e Lambda) per confermare che tutto sia stato cancellato.
